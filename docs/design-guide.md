# 디자인 놀이터 — 디자인 가이드

| 최종 갱신 |
|---|
| 2026-07-15 |

> 사이트 전반의 **텍스트 · 버튼 · 아이콘 · 컬러 · 크기 · 간격** 스펙의 단일 기준.
>
> - **1장(타이포그래피) · 2장(컴포넌트)은 공통 토큰의 단일 원천**이다. 공통값은 반드시 토큰으로 쓰고, 다른 장에서 공통값의 px를 다시 적지 않는다.
> - **페이지 고유 레이아웃 값은 해당 페이지 장에서 정의할 수 있다**(0장 값 분류 참조).
> - 전역 스타일은 공용 `assets/dinol.css` 한 곳에서만 수정한다(전 브리핑·template이 `../../../assets/dinol.css`를 참조 → 자동 소급). 동작 로직은 `assets/dinol.js`.
> - 예외: 운세(`fortune.html`)는 코너 전용 UI라 자체 `<style>`을 쓴다.
> - 변경 이력은 **git이 관리한다.** 이 문서에는 최종 스펙만 적는다.

## 0. 값을 다루는 원칙

> 이 장이 **가장 먼저 적용되는 규칙**이다. 토큰을 쓸지 말지부터 정한다.

### 0-1. 모든 수치는 셋 중 하나다

| 분류 | 정의 | 처리 | 예 |
|---|---|---|---|
| **공통값** | 여러 화면·컴포넌트에서 같은 의미로 반복 | **토큰 필수** (1·2장) | 본문 크기, 기본 패딩, 버튼 높이, Radius |
| **페이지 고유값** | 그 화면의 구조에서만 의미 있는 레이아웃 값 | **로컬 px 허용** — 해당 페이지 장에 명시 | 별자리 카드 이미지 폭 96px, 드로어 폭 480px, 그리드 max-width |
| **예외값** | 광학 보정·미세 조정 | **로컬 px 허용** — 컴포넌트 내부로 한정 | 게이지 두께, 아이콘 선 두께, 텍스트 밀착 간격 |

### 0-2. ⛔ 스케일에 없는 값을 비슷한 토큰에 끼워맞추지 않는다

**가장 위험한 실수다.** 값이 스케일에 없다고 근처 토큰으로 치환하면 화면이 조용히 깨진다.

```css
/* ✗ 절대 금지 — 96px이 스케일에 없다고 간격 토큰(48px)으로 대체 → 폭이 절반이 됨 */
.z-illust{ flex:0 0 var(--space-12); }

/* ✓ 페이지 고유값이면 로컬 px로 두고 문서에 명시 */
.z-illust{ flex:0 0 96px; }  /* 페이지 고유 레이아웃 값 */
```

- **토큰은 의미가 같을 때만 쓴다.** 숫자가 비슷하다는 이유로 쓰지 않는다.
- **범주를 지킨다**: 간격 토큰(`--space-*`)을 크기(width·height·flex)에 쓰지 않고, 크기 토큰(`--size-*`)을 간격에 쓰지 않는다.
- 스케일에 없는 공통값이 필요하면 → **토큰을 억지로 재사용하지 말고 1·2장에 단계를 추가**한다(단계 수 상한 확인).

### 0-3. 토큰 승격

로컬 값이 **2곳 이상에서 같은 의미와 역할로 반복되고, 앞으로 함께 변경될 값**이면 공통 토큰으로 승격한다. 승격 시 스케일 안의 값으로 정렬한다.
**값이 같아도 역할이 다르거나 각자 따로 바뀔 값은 페이지 고유값으로 유지한다**(드로어 폭 480px과 콘텐츠 폭 480px은 우연히 같을 뿐이다).

### 0-4. 스케일 토큰 vs 역할 토큰

토큰이라고 다 같지 않다. **소비처가 없을 때의 처리가 다르다.**

| 종류 | 예 | 미사용 시 |
|---|---|---|
| **스케일(사다리)** | `--step-*` `--fs-*` `--space-*` `--size-*` `--icon-*` `--radius-*` | **유지**. 사다리는 통째로 정의한다 — 한 단계를 빼면 체계가 깨지고 다음 화면에서 다시 만들게 된다 |
| **개별 역할** | `--color-icon-accent` `--color-text-disabled` 등 | **정의하지 않는다**. 소비처가 생길 때 만들고, 그전엔 전환표에 "전환 시 생성"으로만 남긴다 |

- **프리미티브 팔레트(4-2)는 전역 사전**이다. 문서가 원천이고, 각 파일 `:root`에는 **그 파일이 실제 소비하는 것만** 둔다.
- ⛔ 소비처 없는 역할 토큰을 미리 만들지 않는다. 죽은 토큰은 다음 사람이 "있으니까 쓰는" 오용을 부른다.

## 1. 타이포그래피 시스템

> **이 장이 폰트 공통값의 단일 원천이다.**
> 다른 장은 **역할 토큰 이름만 참조**한다. 값을 바꿀 때는 이 장의 스케일 표와 CSS 변수 한 곳만 고친다.
> 폰트는 **예외 없이 전부 공통값**이다 — 로컬 px 폰트 크기는 만들지 않는다(0장 분류상 폰트에 '페이지 고유값'은 없다).

### 1-1. 타입 스케일 (10단계)

**`12 / 14 / 16 / 18 / 20 / 24 / 28 / 32 / 36 / 40`**

- 기본 본문 = **16px**(모바일 본문 최소 권장치). 한글은 영문보다 자소 밀도가 높아 **13px 이하는 장문 본문에 쓰지 않는다.**
- **최소 크기 12px.** 10px 이하 금지(배지·그래프 축 등 극히 예외적인 경우만).
- 18px은 16→20 사이의 실무 보완 단계(카드 제목·팝업 제목·모바일 화면 제목).
- 단계는 **10개가 상한**이다. 새 크기를 추가하려면 이 장을 먼저 고친다.
- 상단(24~40)은 4px 간격, 하단(12~20)은 2px 간격이다. 제목 구간이 촘촘해야 **모바일에서 한 칸만 내려도 급격히 작아지지 않는다**(1-3 모바일 규칙).

### 1-2. 프리미티브 — 스케일 단계 (크기/행간 쌍)

**행간은 배수가 아니라 정수 px로 고정한다.** 배수를 쓰면 `32×1.2=38.4px`처럼 그리드에서 어긋나는 값이 생겨 Figma·CSS 양쪽 관리가 어긋난다. 아래 10단계는 **행간이 4px 그리드에 정렬**된다. 예외는 26px 두 곳(`--step-18` · `--step-16`) — **한글 본문은 행간 여유가 더 필요하고**(자소 밀도가 라틴보다 높음), 사이트 기존 본문 표준도 1.6이라 이에 맞춘다. 24px(150%)은 WCAG 최소선이라 한글 장문엔 빡빡하다.

| 단계 토큰 | 크기 / 행간 | 비율 |
|---|---:|---:|
| `--step-40` | 40 / 48px | 120% |
| `--step-36` | 36 / 44px | 122% |
| `--step-32` | 32 / 40px | 125% |
| `--step-28` | 28 / 36px | 129% |
| `--step-24` | 24 / 32px | 133% |
| `--step-20` | 20 / 28px | 140% |
| `--step-18` | 18 / 26px | 144% |
| `--step-16` | 16 / 26px | 163% |
| `--step-14` | 14 / 20px | 143% |
| `--step-12` | 12 / 16px | 133% |

> 단계는 **의미를 담지 않는다.** `--step-16`은 "본문"이 아니라 단지 시스템에 존재하는 값이다. 의미는 1-3의 역할 토큰이 부여한다. (프리미티브가 의미를 품기 시작하면 시스템이 경직된다.)

### 1-3. 시맨틱 — 역할 토큰

**브레이크포인트: 모바일 = 화면 폭 ≤ 580px**

| 토큰 (CSS 변수) | 역할 | PC 단계 | MO 단계 | PC | MO | 기본 굵기 | 권장 용도 |
|---|---|---|---|---:|---:|---:|---|
| `--fs-display` | Display | step-40 | step-36 | 40/48 | 36/44 | 700 | 핵심 메시지·히어로·코너명 |
| `--fs-h1` | Heading 1 | step-32 | step-28 | 32/40 | 28/36 | 700 | 페이지 대표 제목 |
| `--fs-h2` | Heading 2 | step-24 | step-20 | 24/32 | 20/28 | 700 | 주요 섹션 제목 |
| `--fs-h3` | Heading 3 | step-20 | step-18 | 20/28 | 18/26 | 600 | 카드·패널·하위 섹션 제목 |
| `--fs-title` | Title | step-18 | step-18 | 18/26 | 18/26 | 500 | 리스트·다이얼로그·컴포넌트 제목 |
| `--fs-body` | Body | step-16 | step-16 | 16/26 | 16/26 | 400 | 기본 본문·입력값 |
| `--fs-body-sm` | Body Small | step-14 | step-14 | 14/20 | 14/20 | 400 | 보조 설명·폼 라벨 |
| `--fs-caption` | Caption | step-12 | step-12 | 12/16 | 12/16 | 400 | 날짜·메타 정보·범례 |

**모바일 규칙 — "제목류는 한 칸 내린다"**
- **Display·H1·H2·H3만 스케일에서 한 단계 아래로 이동.** Title 이하 본문류는 PC·MO 동일.
- 크기·행간을 따로 튜닝하지 않는다. **한 칸 내리면 행간은 자동으로 따라온다**(MO Display = `step-36`). → **모바일 전용 값이 없으므로 스케일이 두 벌로 갈라지지 않는다.**
- ⚠️ 축소는 **줄바꿈 방지**가 목적이다. 실측 없이 "작아 보인다"·"커 보인다"로 조정하지 않는다. **먼저 `canvas.measureText`로 최장 문자열의 필요 폭과 가용 폭을 재고**, 여유가 크면 축소 폭을 줄인다(스케일에 단계를 추가해 해결하지, 모바일 전용 값을 만들지 않는다).
- ⚠️ 기존 규칙 "MO가 PC보다 1~2px 큼"은 **폐기**. PC 본문이 15px이라 모바일에서 16px를 확보하려던 우회책이었고, 이제 본문 자체가 16px이라 불필요하다.

**같은 크기는 굵기·컬러로 구분한다** (크기를 새로 만들지 않는다)

| 예시 | 크기 | 굵기 | 용도 |
|---|---:|---:|---|
| Body / Bold | 16 | 700 | 수치 강조(퍼센트 등) |
| Body / Semibold | 16 | 600 | 섹션 라벨·강조 본문 |
| Body / Regular | 16 | 400 | 일반 본문 |

⚠️ **H3 · Title 충돌 주의**: 모바일에서 둘 다 `step-18`(18/26)로 겹친다. 같은 화면에 나란히 쓸 때는 **굵기 차이를 200 이상** 두거나 컬러로 구분한다(기본값 H3 600 vs Title 500은 차이가 작음). 애초에 "한 화면 3~4개 원칙"을 지키면 충돌할 일이 거의 없다.

**한 화면 원칙**: 시스템에 8단계가 있어도 **한 화면에서 실제로 보이는 크기는 3~4개 이내**로 제한한다(제목·본문·보조·캡션).

### 1-4. CSS 구현

```css
:root{
  /* 프리미티브: 스케일 단계 (의미 없음) */
  --step-40-fs:40px; --step-40-lh:48px;
  --step-36-fs:36px; --step-36-lh:44px;
  --step-32-fs:32px; --step-32-lh:40px;
  --step-28-fs:28px; --step-28-lh:36px;
  --step-24-fs:24px; --step-24-lh:32px;
  --step-20-fs:20px; --step-20-lh:28px;
  --step-18-fs:18px; --step-18-lh:26px;
  --step-16-fs:16px; --step-16-lh:26px;
  --step-14-fs:14px; --step-14-lh:20px;
  --step-12-fs:12px; --step-12-lh:16px;

  /* 시맨틱: 역할 → 단계 */
  --fs-display:var(--step-40-fs); --lh-display:var(--step-40-lh);
  --fs-h1:var(--step-32-fs);      --lh-h1:var(--step-32-lh);
  --fs-h2:var(--step-24-fs);      --lh-h2:var(--step-24-lh);
  --fs-h3:var(--step-20-fs);      --lh-h3:var(--step-20-lh);
  --fs-title:var(--step-18-fs);   --lh-title:var(--step-18-lh);
  --fs-body:var(--step-16-fs);    --lh-body:var(--step-16-lh);
  --fs-body-sm:var(--step-14-fs); --lh-body-sm:var(--step-14-lh);
  --fs-caption:var(--step-12-fs); --lh-caption:var(--step-12-lh);
}
/* 모바일: 제목류만 한 칸 내림 (본문류는 그대로) */
@media (max-width:580px){
  :root{
    --fs-display:var(--step-36-fs); --lh-display:var(--step-36-lh);
    --fs-h1:var(--step-28-fs);      --lh-h1:var(--step-28-lh);
    --fs-h2:var(--step-20-fs);      --lh-h2:var(--step-20-lh);
    --fs-h3:var(--step-18-fs);      --lh-h3:var(--step-18-lh);
  }
}
```

**구현 규칙**
- 컴포넌트는 `font-size:var(--fs-body)`처럼 **역할 토큰만 참조**한다. px·단계 토큰 직접 참조 금지.
- 스케일에 없는 크기가 필요해도 **근처 토큰으로 대체하지 않는다**(0-2).
- ⚠️ **크기와 행간은 항상 쌍으로 적용**한다(`--fs-body` + `--lh-body`). 행간이 px 단위라 **상속되면 자식이 자기 크기와 무관한 행간을 물려받는다.** 배수(`1.5`)와 달리 px는 계산값이 그대로 내려가므로, 행간을 생략하지 않는다.
- 요소마다 `@media`를 반복하지 않는다 — 반응형은 위 `:root` 한 곳에서만 처리.

### 1-5. 적용 현황

| 영역 | 상태 | 비고 |
|---|---|---|
| **운세 — 오늘의 별자리** (`fortune.html`) | ✅ **적용 완료** | 11장 참조 |
| 메인 — AI & Design News | ⏳ 전환 예정 | 아래 1-6 레거시 표가 현행 |
| 아카이브 · 개인정보 | ⏳ 전환 예정 | 〃 |
| 디놀 톡톡 (방명록) | ⏳ 전환 예정 | 8·10장이 현행 |
| 팝업(드로어) — 브리핑 카드 | ⏳ 전환 예정 | 5장이 현행 |

> 전환은 단계별로 진행한다. 전환이 끝난 영역은 해당 장의 px 표를 **토큰 이름으로 교체**하고 이 표를 갱신한다.

| | 저작권 `footer-copy` | © 2026 디자인놀이터… | 13px / w600 | 13px | `#888888` |
| | 개인정보 링크 `footer-link` | 개인정보처리방침 | 16px / w400 · 밑줄 | 16px | `#666666` |
| **개인정보** | 사이트 타이틀 `site-title` | 디자인 놀이터 | 18px / w300 | 18px | `#aaaaaa` |
| | 히어로 부제 `site-sub` | 개인정보처리방침 | 32px / w700 | 32px | `#ffffff` |

## 2. 컴포넌트 시스템

> **이 장이 크기·간격 공통값의 단일 원천이다.** (폰트는 1장)
> 단, 크기·간격에는 **페이지 고유값·예외값이 존재한다** → 0장 분류를 먼저 적용한다.

### 2-1. 컴포넌트 높이

| 토큰 | 높이 | 용도 |
|---|---:|---|
| `--size-xs` | 28px | 소형 배지·태그 |
| `--size-s` | 32px | 보조 컨트롤 |
| `--size-m` | 40px | 아이콘 버튼 프레임·일반 컨트롤 |
| `--size-l` | 48px | **기본 버튼·입력창** (디놀 표준) |
| `--size-xl` | 56px | 강조 CTA |

- **디놀 기본은 `--size-l`(48px).** 기존 아카이브·오픈채팅·메인으로·원문·닫기 버튼이 모두 48px이라 이미 일관돼 있다 → 그대로 표준화.
- **버튼과 입력창은 같은 높이 스케일을 공유한다.** 각각 임의 크기로 만들지 않는다.

### 2-2. 터치 타깃 (접근성 · 필수)

> **인터랙티브 요소의 터치 영역은 최소 44×44px**(`--touch-min`). 시각적 외형이 그보다 작아도 여백·의사요소로 영역을 확보한다.

```css
/* 외형은 작게, 터치 영역만 확보 */
.icon-btn{ width:var(--size-m); height:var(--size-m); position:relative; }
.icon-btn::after{ content:""; position:absolute; inset:50%; width:var(--touch-min); height:var(--touch-min); transform:translate(-50%,-50%); }
```

- 아이콘 자체는 16~24px이어도 **버튼 프레임 40px + 터치 44px 이상**이 기본값.
- 체크박스·라디오처럼 작은 컨트롤은 **라벨을 포함한 행 전체**를 클릭 영역으로 잡는다.
- ⚠️ **아이콘 버튼에는 반드시 `aria-label`을 단다.** 아이콘 이름이 아니라 **동작**을 쓴다(`닫기`·`좋아요` ○ / `x-icon` ✕).

### 2-3. 아이콘 크기

| 토큰 | 크기 | 용도 |
|---|---:|---|
| `--icon-s` | 16px | 인라인·보조(아카이브 이동 화살표) |
| `--icon-m` | 20px | **기본** — 버튼·카드 액션(좋아요·공유) |
| `--icon-l` | 24px | 주요 액션·내비게이션 |
| `--icon-xl` | 32px | 빈 화면·큰 카드 |

**핵심 배열 `16 / 20 / 24 / 32`.** 12px 이하는 상태점 수준의 예외만 허용.

### 2-4. 간격 (Spacing) — 4px 그리드

| 토큰 | 값 | 주요 용도 |
|---|---:|---|
| `--space-1` | 4px | 아이콘·텍스트 미세 간격 |
| `--space-2` | 8px | 기본 내부 요소 간격 |
| `--space-3` | 12px | 버튼·칩 내부 |
| `--space-4` | 16px | **기본 패딩** |
| `--space-5` | 20px | 카드·폼 |
| `--space-6` | 24px | 섹션 내부 |
| `--space-8` | 32px | 섹션 간격 |
| `--space-10` | 40px | 큰 그룹 |
| `--space-12` | 48px | 페이지 섹션 |

- **중심 6개 `4 / 8 / 12 / 16 / 24 / 32`** 로 운영하고, 20·40·48은 필요할 때만.
- 페이지 좌우 여백: 모바일 `--space-4`(16px) / 데스크톱 `--space-6`~`--space-8`(24~32px).

**4px 그리드 — 기본 원칙과 예외**

| 대상 | 규칙 |
|---|---|
| 레이아웃·공통 컴포넌트 간격 | **4px 그리드 필수** (`--space-*` 토큰) |
| 광학 보정, 텍스트 밀착 간격, 선·게이지 두께, 아이콘 정렬 | **2px 단위 또는 예외값 허용** |

- 예외값은 **컴포넌트 내부 로컬 값으로 한정**한다. 공통 토큰으로 승격하지 않는다.
- 애매한 값은 그리드로 정렬한다: `3→4` · `7→8` · `11→12` · `18→16 또는 20` · `21→20 또는 24`.

### 2-5. Radius

| 토큰 | 값 | 용도 |
|---|---:|---|
| `--radius-s` | 4px | 소형 상태·태그 |
| `--radius-m` | 8px | **일반 버튼·입력창** |
| `--radius-l` | 12px | 카드·모달 내부 블록 |
| `--radius-xl` | 16px | 큰 카드·바텀시트·드로어 |
| `--radius-full` | 999px | 칩·원형 버튼·게이지 바 |

**핵심 배열 `4 / 8 / 12 / 16 / Full`.** 같은 역할에 6·7·9px을 섞지 않는다.

### 2-6. Border

| 토큰 | 값 | 용도 |
|---|---:|---|
| `--border-thin` | 1px | 입력창·카드·구분선 |
| `--border-strong` | 2px | 포커스·선택·강조 |

> 이미 1px/2px만 쓰고 있어 **변경 없음.** 포커스 링은 `--border-strong` + 포인트 컬러 `#ff7f3e`.

### 2-7. CSS 구현

```css
:root{
  /* 컴포넌트 높이 */
  --size-xs:28px; --size-s:32px; --size-m:40px; --size-l:48px; --size-xl:56px;
  --touch-min:44px;
  /* 아이콘 */
  --icon-s:16px; --icon-m:20px; --icon-l:24px; --icon-xl:32px;
  /* 간격 (4px 그리드) */
  --space-1:4px;  --space-2:8px;  --space-3:12px; --space-4:16px;
  --space-5:20px; --space-6:24px; --space-8:32px; --space-10:40px; --space-12:48px;
  /* Radius */
  --radius-s:4px; --radius-m:8px; --radius-l:12px; --radius-xl:16px; --radius-full:999px;
  /* Border */
  --border-thin:1px; --border-strong:2px;
}
```

- 컴포넌트는 **공통값에 한해** 토큰만 참조한다. 페이지 고유값·예외값은 로컬 px + 주석(0장).
- 크기 토큰은 **반응형으로 바뀌지 않는다**(폰트와 다른 점). 밀도 조정이 필요하면 간격 토큰 단계를 바꿔 쓴다.

### 2-8. 적용 현황

| 영역 | 상태 |
|---|---|
| **운세 — 오늘의 별자리** (`fortune.html`) | ✅ **적용 완료** · 11장 참조 |
| 메인 · 아카이브 · 개인정보 | ⏳ 전환 예정 — 3장 레거시 표가 현행 |
| 디놀 톡톡 · 브리핑 팝업 | ⏳ 전환 예정 |

## 4. 컬러 시스템

> **이 장이 컬러 공통값의 단일 원천이다.** 다른 장은 HEX를 적지 않고 **시맨틱 토큰만 참조**한다.
> 컬러 변경 시 이 장의 프리미티브 팔레트와 시맨틱 토큰만 고친다.

### 4-1. 구조 — 3계층

```
프리미티브(--neutral-500)  →  시맨틱(--color-text-muted)  →  컴포넌트(.z-en)
   HEX만, 용도 없음              의미 부여                    시맨틱만 참조
```

**운영 원칙**
- 컴포넌트에서 **HEX 직접 입력 금지**. 프리미티브 직접 참조도 금지(시맨틱 정의부에서만).
- 같은 HEX라도 **역할이 다르면 시맨틱 토큰을 분리**한다(`--color-text-accent` ≠ `--color-border-focus`).
- 같은 계열의 유사색을 새로 만들지 않는다. 필요한 색은 **기존 팔레트에서 먼저 고른다**.
- 호버 컬러를 컴포넌트마다 새로 만들지 않는다.
- **JS/SVG에서도 HEX를 쓰지 않는다** — CSS 변수를 읽어 쓴다.
  ```js
  const CV = n => getComputedStyle(document.documentElement).getPropertyValue(n).trim();
  line.setAttribute('stroke', CV('--color-border-strong'));
  ```

### 4-2. 프리미티브 팔레트

**Neutral** — 숫자가 클수록 어둡다. **전부 실제 사용 중인 값**이다. Orange도 같은 방향(숫자↑ = 진함)으로 정렬한다.

| `--neutral-350` | `#a9adbe` | 설명 텍스트 |
| `--neutral-400` | `#999999` | 메타 정보 |
| `--neutral-500` | `#8a8a99` | 낮은 강조·아이콘 |
| `--neutral-600` | `#888888` | 낮은 정보 |

| **`--orange-500`** | **`#ff7f3e`** | **디놀 포인트 컬러 (고정)** |
| `--purple-400` | `#8a8aff` | 카테고리·정보 |
| `--red-400` | `#e46a6a` | 삭제·위험 보조 |
| `--red-500` | `#f64444` | 오류 |
| `--kakao-500` | `#fee500` | 카카오 오픈채팅 |
| `--kakao-400` | `#ffe94d` | 카카오 호버 |

> **포인트 컬러 `#ff7f3e`는 디놀 아이덴티티다. 변경하지 않는다.**
> 용도 제한: 현재 페이지 · 좋아요 활성 · 섹션 라벨 · 선택 상태 · 핵심 수치·게이지 · 포커스 링. **넓은 배경 면에는 쓰지 않는다.**
> Purple은 Orange와 경쟁하지 않게 카테고리·정보성 보조로만 제한한다.

> **4-3 ~ 4-5의 시맨틱 표는 사이트 전체의 '역할 사전'이다.**
> 각 파일의 `:root`에는 **그 파일이 실제 소비하는 토큰만** 정의한다(0-4).
> 4-9에서 "전환 시 생성"으로 표시된 역할은 **운영 CSS에 아직 존재하지 않는다** — 사전에 적혀 있다고 바로 쓰면 안 된다.

### 4-3. 시맨틱 — 배경·표면

명도 사다리: `base < sunken < surface < elevated < raised < secondary < hover`

| `--color-bg-sunken` | `--neutral-985` | 함몰 블록(관계도 등) |

| `--color-bg-hover` | `--neutral-800` | 표면 호버 |

| `--color-bg-disabled` | `--neutral-825` | 비활성·플레이스홀더 |

### 4-6. 버튼 상태  ⏳ 전환 예정

> 버튼은 아직 토큰 전환 전이다(3-3 레거시 표가 현행). 아래는 **전환 시 적용할 스펙**이다.
> 밝은 면 버튼(아카이브·오픈채팅·메인으로·톡톡 등록)이 **4곳에서 같은 역할로 반복**되므로, 전환 시 컴포넌트 토큰으로 승격한다(0-3 기준 충족).

**전환 시 생성할 컴포넌트 토큰**
```css
--color-button-primary-bg:var(--neutral-200);          /* #c9ccd4 */
--color-button-primary-bg-hover:var(--neutral-150);    /* #d9dce4 */
--color-button-primary-bg-pressed:var(--neutral-300);
--color-button-primary-text:var(--ink-900);
```

| 상태 | Primary(밝은 면) | Secondary(어두운 면) | Accent(좋아요·현재) |
|---|---|---|---|
| Default | `--color-button-primary-bg` / `--color-button-primary-text` | `--color-bg-secondary` / `--color-text-secondary` | `--color-icon-secondary` |
| Hover | `--color-button-primary-bg-hover` | `--color-bg-hover` / `--color-text-primary` | `--color-text-accent-soft` |
| Pressed | `--color-button-primary-bg-pressed` | `--color-border-default` | `--color-icon-accent` |
| Focus | Default + `--color-border-focus` 링 | Default + 링 | Default + 링 |
| Disabled | `--color-bg-disabled` / `--color-text-disabled` | 〃 | `--color-text-disabled` |

- **Destructive**는 강조 버튼으로 만들지 않는다 — Text/Tertiary 형태 + `--color-icon-danger`.
- 포인트 버튼(톡톡 등록 `#ff7f3e`) 호버는 `--orange-450`(`#ff9560`) — **base보다 밝아지는 게 의도된 방향**이다.
- 카카오 버튼은 `--kakao-500` → 호버 `--kakao-400`. 일반 Primary에 브랜드 컬러를 쓰지 않는다.
- ⚠️ 표의 `--color-icon-*` · `--color-text-disabled` · `--color-button-*`는 **아직 정의하지 않는다**(0-4). 해당 영역 전환 시 생성한다.

### 4-7. 접근성

**텍스트 대비 실측표** (WCAG 일반 4.5:1 / 큰 텍스트 3:1)

| 텍스트 토큰 | `base` | `sunken` | `surface` | `elevated` | `raised` | `secondary` | `hover` |
|---|---|---|---|---|---|---|---|
| `--color-text-primary` | 19.38 | 19.21 | 18.78 | 18.44 | 17.45 | 15.54 | 13.74 |
| `--color-text-body` | 12.54 | 12.43 | 12.16 | 11.94 | 11.29 | 10.06 | 8.89 |
| `--color-text-secondary` | 9.03 | 8.95 | 8.75 | 8.59 | 8.13 | 7.24 | 6.40 |
| `--color-text-tertiary` | 8.68 | 8.61 | 8.41 | 8.26 | 7.82 | 6.96 | 6.16 |
| `--color-text-meta` | 6.80 | 6.74 | 6.59 | 6.47 | 6.12 | 5.45 | 4.82 |
| `--color-text-muted` | 5.70 | 5.65 | 5.52 | 5.42 | 5.13 | 4.57 | 4.04 ✗ |
| `--color-text-disabled` | 3.38 ✗ | 3.35 ✗ | 3.27 ✗ | 3.21 ✗ | 3.04 ✗ | 2.71 ✗ | 2.39 ✗ |

**허용 조합 규칙** — "모든 조합이 통과"가 아니다. 아래를 지킬 때만 기준을 만족한다.

- 일반 콘텐츠 텍스트(`primary` `body` `secondary` `tertiary` `meta`)는 **위 모든 표면에서 4.5:1 이상** — 자유롭게 조합 가능.
- **`--color-text-muted`는 `base` · `sunken` · `surface` · `elevated` · `raised`에만 사용한다.** `secondary`(4.57)는 기준을 통과하나 여유가 작아 가능하면 `meta` 이상을 쓰고, **`hover`(4.04)는 기준 미달이라 금지** → 그보다 밝은 표면에서는 `--color-text-meta` 이상을 쓴다.
- **`--color-text-disabled`는 어떤 표면에서도 4.5:1을 만족하지 않는다**(2.39~3.38). **비활성 컨트롤 전용**이며 본문·안내·필수 정보 전달에 쓰지 않는다. 비활성은 색 외에 **동작 차단 + `not-allowed` 커서**를 함께 제공한다.
- 큰 텍스트(`--fs-h2` 이상)·아이콘·입력 경계는 3:1 이상.

**컬러 단독 사용 금지** — 상태를 색으로만 구분하지 않는다.

| 상태 | 색 + 함께 쓸 단서 |
|---|---|
| 오류 | Red + 오류 문구 + 아이콘 |
| 선택 | Orange + 굵기 또는 인디케이터 |
| 현재 페이지 | Orange + 밑줄 + 굵기 |
| 비활성 | 낮은 명도 + 동작 차단 + 커서 |

**포커스** — 항상 보이게: `outline: var(--border-strong) solid var(--color-border-focus); outline-offset: var(--space-1);`

- **"내용" 컬러**: 필드 본문·인사이트 불릿·코멘트 본문 모두 `#cfcfda`. 방명록 본문 `.gb-entry-body`만 `#a6a6b8`(별개 영역)
- 하단 툴바 버튼 비율 **닫기 : 원문 = flex 3 : 7**. 원문 `drawer-cta`(flex 7) 16px/w400 · 닫기 `drawer-close`(flex 3) 14px/w500 (스타일은 3·9장 참조)

**데이터 규칙 (카드 `data-*`)**
- 공통: `data-category`
- 기본 필드: `data-summary`(한 줄 요약) · `data-points`(핵심 인사이트, `|`구분) · `data-designer` · `data-impact-score`(1~5 정수 별점 = 실무 영향도, 텍스트 `data-impact`는 미사용) · `data-recommend` · `data-comment`(큐레이션 노트 — **한글 카드 단일 / 영문 카드는 `data-comment`(영문)+`data-comment-kr`(한국어) 토글 전환**)
- 영문 카드: 위 기본(영문) + `-kr`(한국어) 각각 + `data-title-en`(영어 원제목, 토글 유발). **카드 제목·썸네일 라벨은 한글로 표시**하고 영어 원제목은 `data-title-en`에 보존 → 팝업 토글에서 노출. 한글 카드는 `data-title-en` 없음(토글 안 뜸)

## 6. 카드 썸네일 — 라벨 언어 정책

카드 썸네일의 짧은 키워드(`thumb-label`)는 **영문·한글 기사 모두 한글**로 쓴다 (예: `정책` · `이미지 생성` · `리브랜딩` · `특화 AI`).
- 영문 출처 표시는 **EN 배지**가 담당하므로 라벨까지 영어일 필요가 없다.
- 이유: 스크롤하며 0.5초 안에 주제를 파악하는 **스캔용 라벨**이라 모국어(한글)일 때 인지가 가장 빠르다. 브랜드·약어는 자연스러우면 한글(피그마·미스트랄), `AI`처럼 관용 표기는 그대로 둔다.
- 폰트: **산세리프 · w500 · 자간 1px** (세리프+넓은 자간은 라틴 대문자용이라 한글엔 부적합).

## 11. 운세 코너 — 오늘의 별자리 (`fortune.html`)

> ✅ 1·2장 토큰 적용 완료 영역. **이 장에는 px를 적지 않는다** — 어느 요소에 어떤 토큰을 쓰는가만 정의한다.

### 11-1. 메인

**헤더**

### 11-2. 상세 팝업(드로어)

**규격**: PC **560px** 중앙 모달 / MO 100% 바텀시트, 세로 90vh. 스크롤바 폭 6px · thumb `--color-border-default`.

**레이아웃 순서**: 심볼 이미지 → 한글 별자리명 → 영문 별자리명(서브) → 진짜 별자리 운세 → 오늘의 확률 → 오늘의 폰트·컬러 → 오늘의 궁합

- **별자리명에 럭키폰트를 적용하지 않는다.** 기본 산세리프 `--color-text-primary` 고정 → 럭키폰트는 "오늘의 기운" 블록 `Aa` 아이콘에서만 시각화.
- 심볼 박스(`.d-symbox`): **페이지 고유값** PC 88px / MO 80px 정사각, 회색 `--color-bg-disabled`(이미지 추가 전 자리).

**오늘의 확률 — 항목 구조** (한 항목 = 3행)

```
[아이콘 항목명 ················· 88% · 번뜩임✨]   ← .prob-head (양끝 정렬)
[설명 문구]                                      ← .prob-quip (항목명과 같은 기준선)
[━━━━━━━━━━━━━━━━━━━━━]                          ← .prob-bar (전체 폭)
```

- **항목명은 절대 줄바꿈하지 않는다**(`white-space:nowrap`). 좌측 고정폭 열을 두면 `수정 굴레 피할 운` 같은 라벨이 2줄로 깨져 행 높이가 들쭉날쭉해진다.
- 설명은 **항목명과 같은 기준선**(들여쓰기 없음) — 설명이 게이지가 아니라 항목에 속함을 근접성으로 드러낸다.
- 게이지는 **전체 폭**을 쓴다. 폭이 좁으면 40%와 88%의 길이 차이가 안 보인다.
- 퍼센트·상태는 `·`로 묶어 **하나의 결과 그룹**으로 우측 정렬.
- 게이지 바: 높이 8px(예외값) · 트랙 `--color-bg-track` · 활성 `linear-gradient(90deg, var(--color-accent-grad-from), var(--color-accent-grad-to))` · `--radius-full`
- "오늘의 기운"은 폰트·컬러 2열 그리드(MO 1열).
