## 1. 목표 · 포지셔닝

- 디놀의 새 코너: **디자이너를 위한 별자리 운세**. (디놀 = 상위 브랜드, 운세는 하위 코너)
- 컨셉: **실제 별자리 운세 데이터를 받아와 → 디자이너 버전으로 재해석**.
- ⚠️ "오하아사(おはよう朝日)"는 특정 방송 프로그램명이라 그대로 쓰면 상표·제휴 오해 소지 → **디놀 고유 이름으로** 갈 것(아래 미정 참고). 오하아사는 "영감받은 포맷"으로만.
- ⚠️ **순위(1~12위 랭킹)는 넣지 않기로 결정.** 이유: 오하아사 유명세는 "순위 적중 소문"이라는 서사에서 나오는데 그 서사를 빌릴 수 없고, 순위(범용운세)가 디자이너 특화 콘텐츠와 안 맞음. 방송사·사이트마다 순위가 다 달라 "정답 순위"도 없음.

## 2. 확정된 화면 설계 (2층 구조)

### 기본 카드 = 진짜 별자리 운세 (그대로)
- freehoroscopeapi에서 가져온 **실제 운세를 한국어 번역**해서 표시.
- 항목: 별자리명·기호 · **총점(별점 ★)** · **메시지(번역된 실제 운세)** · [디자이너 운세 →] 버튼
- 정렬: **별자리 순서**(양자리→…→물고기자리). 순위 없음.

### 자세히 보기 → 팝업(드로어) = 디자이너 운세 (우리 재해석)
- 디놀 톡톡 드로어와 같은 팝업 패턴(UX 일관).
- 항목: **확률 6종**(야근·외주·한번에통과·3회+수정·주목/채택·리젝) · **궁합 6종**(팀장·동료디자이너·기획자·개발자·대표·클라이언트) · **기운**(행운의 색 HEX+이름·행운의 폰트)
- **팝업 타이틀(별자리명)을 그 별자리의 행운의 폰트 + 행운의 색으로 꾸밈** ← 시그니처.
- 공유·닫기 버튼.

### 와이어프레임
```
[기본] 12별자리 리스트 (순위 없음, 별자리 순서)
┌─────────────────────────────┐
│ ♒ 물병자리          ★★★★☆   │
│  "번역된 실제 운세 메시지…"    │
│              [디자이너 운세 →] │
├─────────────────────────────┤
│ ♓ 물고기자리        ★★★☆☆   │
│  ...                         │
└─────────────────────────────┘
        │ [디자이너 운세 →]
        ▼
┌─ 팝업 (행운의 폰트·색 타이틀) ──┐
│  ♒ 물병자리 · 디자이너 운세     │
│  📊 확률 6  🤝 궁합 6  🎨 기운  │
│  [공유]           [닫기]      │
└──────────────────────────────┘
```

## 3. 확정된 데이터 흐름 (핵심 아키텍처)

```
① GitHub Action (매일 새벽 cron) — Claude 루틴 아님!
     → freehoroscopeapi에서 12별자리 실제 운세 fetch (외부 fetch 가능)
     → horoscope_raw.json 저장(커밋)

② Claude 루틴 (매일 아침, 이미 브리핑용으로 돎)
     → horoscope_raw.json 읽기 (raw.githubusercontent 허용됨)
     → 각 별자리: 한국어 번역 + 총점 별점 산정 + 디자이너 재해석(확률·궁합·색·폰트)
     → fortune.json 저장

③ fortune.html (프론트)
     → fortune.json 읽어 렌더 (같은 사이트 = CORS·키·비용 전부 없음)
     → 기본 카드 = 번역 운세+별점, 팝업 = 디자이너 재해석
```

**이 구조가 왜 필요한가(블로커 해결):**
| 블로커 | 해결 |
|---|---|
| Claude 루틴은 외부 fetch 403 | GitHub Action이 대신 fetch |
| 브라우저 직접 호출 시 CORS | 프론트는 같은 사이트 json만 읽음 |
| API 키 노출 | (키 필요시) GitHub Secrets |
| 저작권(원문 재게시) | 번역+재해석 = 변형저작물, 원문 그대로 안 씀 |
| DB 없음 | json 파일이면 충분(index.json처럼) |

## 4. 데이터 소스 — freehoroscopeapi (확정)

- 엔드포인트: `https://freehoroscopeapi.com/api/v1/get-horoscope/daily?sign=aries`
- HTTPS ✅ · 키 불필요 ✅ · 12별자리 · daily/weekly/monthly
- 응답: `{"data":{"date":"2026-07-10","period":"daily","sign":"Aries","horoscope":"영어 운세 텍스트"}}`
- **주는 것: 운세 텍스트(영어)뿐.** 별점·순위·럭키컬러 없음 → 별점·색·폰트·확률·궁합은 **루틴이 생성**.
- 2026-07-10 기준 정상 응답 확인됨(사용자 브라우저 테스트).
- 리스크: 취미성 오픈 프로젝트라 안정성 불확실 → **API 실패 시 폴백**(직전 json 유지 또는 date-seeded) 필요.
- sign 파라미터(영문): aries, taurus, gemini, cancer, leo, virgo, libra, scorpio, sagittarius, capricorn, aquarius, pisces

## 5. 개발 순서 (화면 확정됐으니 이 순서로)

1. **fortune.json 스키마 확정** — 화면 항목에 맞춰. 초안:
```json
{
  "date": "20260710",
  "signs": [
    {
      "sign": "aquarius", "nameKo": "물병자리", "symbol": "♒",
      "score": 4,                         // 총점 별점 1~5 (루틴이 긍정도로 산정)
      "message": "한국어로 번역된 실제 운세…", // 기본 카드
      "designer": {                        // 팝업(디자이너 재해석)
        "probs": [ {"label":"야근","icon":"🌙","pct":60,"quip":"…"}, … 6개 ],
        "matches": [ {"name":"개발자","emoji":"💻","stars":5,"quip":"…"}, … 6개 ],
        "luckyColor": {"hex":"#6B5B95","name":"트와일라잇 뮤즈"},
        "luckyFont": "Gaegu"              // 구글폰트 패밀리명 (아래 폰트풀에서)
      }
    }, … 12별자리 (별자리 순서)
  ]
}
```
2. **GitHub Action** — `.github/workflows/horoscope.yml`. cron으로 매일 12별자리 fetch → horoscope_raw.json 커밋. (Action은 외부 fetch 가능)
3. **fortune.html** — fortune.json 읽어 리스트+팝업 렌더. (아래 기존 v1 참고/재작업)
4. **루틴 지침** — routine_instruction.md + 루틴 UI에 "운세 재해석" 단계 추가: horoscope_raw.json 읽고 → 번역·별점·확률·궁합·색·폰트 생성 → fortune.json 작성.

## 10. 별자리 일러스트 배치 (2026-07-14 추가) 🎨

- **에셋**: 12별자리 캐릭터 일러스트를 `assets/zodiac/<sign>.webp`로 배치. (파일명 = 데이터 `sign` 값, 영문 소문자 — 파일명 정책 준수)
  - aries · taurus · gemini · cancer · leo · virgo · libra · scorpio · sagittarius · capricorn · aquarius · pisces
- **최적화**: 원본 PNG 1024×1536(각 ~2.2MB, 합 ~27MB) → WebP 640px, quality 80으로 변환. **합계 ~910KB(각 ~75KB)**. 웹 로드 부담 제거.
- **메인 카드 레이아웃 변경**: 기존 "이모지 원형 배지 + 세로 스택" → **가로 레이아웃**으로 변경.
  - 왼쪽: 세로 일러스트 스트립(`.z-illust`, flex 0 0 118px). 박스 비율이 원본 2:3에 근접해 캐릭터 전신이 잘림 최소로 노출. `object-position:50% 12%`.
  - 오른쪽: `.z-body`(이름·메시지·CTA).
  - 이모지 기호(`.z-sym`)는 일러스트 좌하단 반투명 원형 배지(30px)로 이동 — 기호 모티프 유지.
  - hover 시 일러스트 살짝 확대(scale 1.05).
- **폴백**: `<img onerror>`로 이미지 로드 실패 시 이미지 숨김 → `.z-illust` 그라데이션 배경 + 이모지 배지만 노출(깨진 카드 방지).
- **드로어(팝업)**: 이번 변경 범위 밖(메인만). 필요 시 드로어 헤드에도 동일 일러스트 추가 가능.
- ⚠️ 세로형 전신 일러스트라 **가로 히어로 밴드 크롭은 부적합**(캐릭터 잘림) — 세로 스트립 방식 유지 권장.
