## 6. 카드 썸네일 — 라벨 언어 정책 (2026-07-06 전면 한글화)

카드 썸네일의 짧은 키워드(`thumb-label`)는 **영문·한글 기사 모두 한글**로 쓴다 (예: `정책` · `이미지 생성` · `리브랜딩` · `특화 AI`).
- 영문 출처 표시는 **EN 배지**가 담당하므로 라벨까지 영어일 필요가 없다.
- 이유: 스크롤하며 0.5초 안에 주제를 파악하는 **스캔용 라벨**이라 모국어(한글)일 때 인지가 가장 빠르다. 브랜드·약어는 자연스러우면 한글(피그마·미스트랄), `AI`처럼 관용 표기는 그대로 둔다.
- 폰트: **산세리프 · w500 · 자간 1px** (세리프+넓은 자간은 라틴 대문자용이라 한글엔 부적합).

## 10. 디놀 톡톡 댓글 (2026-07-07)

방명록 글의 서브컬렉션 `guestbook/{글}/comments`에 저장. 비밀번호(4자리)·수정/삭제는 방명록과 **동일한 클라이언트 검증**(SHA-256 해시 대조 + 마스터 `481516`).

- **댓글 N 토글**: 카드 하단, 포인트 주황 `#FF7F3E`. 탭하면 **지연 로딩**(펼칠 때만 읽음). 접힘 ▼ / 펼침 ▲(`.gb-cchev.up` rotate180).
- **댓글 아이템**(`.gb-comment`, `#15151f` 칩): 이모지 아바타 + 닉네임(13px) + 시간 + ⋯ + 본문(padding-left 23px).
- **옵션 버튼 ⋯**(`.gb-kebab`): 글·댓글 **동일 크기·색**(18px·`#6c6c7a`). 탭 → 팝업(`.gb-menu`, 수정 `#dcdce4` / 삭제 `#e46a6a`), 글·댓글 동일 스타일. **기존 상시노출 수정/삭제 버튼을 대체**.
- **입력 폼**(`.gb-cform`): 닉네임(분야)+비밀번호 4자리 한 줄 → 댓글(**100자**, 입력칸 우하단 카운터 `.gb-ccount`) → 얇은 스마일(선 1.4)+등록(주황 텍스트). 스마일 클릭 시 **이모지 피커**. **점진적 노출(2026-07-07)**: 댓글 영역을 펼치면 기본은 `.gb-cadd` **'댓글 달기' 트리거 버튼만** 보이고, 클릭해야 입력 폼이 펼쳐진다. 폼 하단 바 = 스마일(좌) + `.gb-cactions`(취소·등록, 우). **취소**(`.gb-ccancel`) 또는 등록 성공 시 폼이 다시 접힌다(`composerOpen` 상태). 이모지 40개(8×5).
- **댓글 아이콘**: 2.5D 흰 말풍선(점3개) + 뒤 포인트 컬러(보라 `#6f66cc`) 레이어 — 인라인 SVG.
- 모바일 탭 영역 **44px**(투명 padding + 음수 margin, 배경 없음).
- 글 문서에 **`commentCount`** 필드(댓글 추가/삭제 시 `increment`) → "댓글 N"을 추가 읽기 0으로 표시.
- **주의**: 글 삭제 시 하위 댓글은 Firestore가 자동 삭제하지 않음(고아 문서 잔존, UI엔 안 보임).
- 보안 규칙: `firestore.rules`에 `comments` 서브컬렉션 + `commentCount` 갱신 규칙 추가(App Check는 콘솔에서 이미 강제).
